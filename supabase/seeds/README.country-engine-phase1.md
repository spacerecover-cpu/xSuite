# Country Engine Phase 1 — Operator-Apply Runbook (Area 2: geo + onboarding)

These `*.operator.sql` files are migrations the **operator** applies. The subagent
that authored Area 2 is barred (by task constraints) from applying migrations,
regenerating `database.types.ts`, deploying edge functions, or installing deps.
This runbook lists what to apply, in order, and what to do after each.

## Apply order

1. **`populate_geo_countries_reference_data.operator.sql`**
   - Adds `geo_countries.{source_locked,data_source,source_version}` (NOT present
     on the live table — verified 2026-06-15), runs the generated seed
     (`geo_countries_seed.generated.sql`), then de-activates every country not in
     the hand-verified set (`SA AE OM KW QA BH GB IN US`).
   - The seed body is generated by `npm run geo:build-seed`
     (`scripts/country-engine/build-geo-seed.ts` + `gcc-priority-seed.ts`).
   - After: `npm run db:types`; add a `migrations.manifest.md` row; `check-schema-drift`; `check-tsc` → 0.
   - Then **flip the CI gate to enforcing**: remove `continue-on-error: true` from
     the `country-config-completeness` job in `.github/workflows/ci.yml` (the
     strict `check-geo-completeness.sql` requires phone/address, which only exist
     post-seed).

2. **`signup_otps_consumed_at.operator.sql`**
   - Adds `signup_otps.consumed_at` for single-use OTP enforcement.
   - After: `npm run db:types`; manifest row; drift; tsc.
   - Then update `provision-tenant/index.ts` at the `TODO(consumed_at)` marker
     (`.is('consumed_at', null)` + stamp on use).

3. **`backfill_seed_existing_tenants.operator.sql`**
   - Creates the missing `onboarding_progress` rows for the 2 live tenants
     (0 → 2). Entity + MAIN branch already exist (auto-collapse migration).
   - Verify: `SELECT count(*) FROM onboarding_progress;` → 2.

## OWNER DECISIONS (not operator-mechanical — see PR blockers)

- **Full ~195-country dataset + library deps.** Area 2 ships an accurate
  hand-verified seed for GCC-6 + GB/IN/US ONLY. Populating ~195 countries needs
  CLDR / ISO 4217 / libphonenumber-js / i18n-postal-address (+ `tsx` to run the
  generator). Those deps were intentionally NOT added. Decide: approve the deps
  + dataset sourcing + a named holiday/refresh owner (per Q1), then extend
  `gcc-priority-seed.ts` (or swap it for a CLDR loader behind the same
  `buildCountryConfigRow` transform) and re-run `npm run geo:build-seed`.

- **Edge-function deploy.** `provision-tenant` was edited + committed but NOT
  deployed (deploy is deferred). Deploy it after migrations 1–2 land.

- **`seed_new_tenant` / `seed_number_sequences` RPCs.** The plan assumes these
  program-track RPCs exist; they do NOT in the live DB. The edge fn + backfill
  create the entity/onboarding rows directly instead. If the RPCs land later,
  swap the inline inserts for `PERFORM seed_new_tenant(id)`.
